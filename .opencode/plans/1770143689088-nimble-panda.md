# Plan: Upgrade dxd-cdn with Marketing CDN Features

## Summary

Migrate all new features from `marketing-cdn` to `dxd-cdn` while:
1. Keeping the existing GitHub proxy + R2 fallback architecture in dxd-cdn
2. Adding all new marketing-cdn features (upload, browse, analytics, file management)
3. Replacing all Point/PDC branding with DXD branding
4. Using the existing `dxd-cdn` R2 bucket

## Key Differences Between Codebases

| Aspect | dxd-cdn (Current) | marketing-cdn (Source) |
|--------|-------------------|------------------------|
| **Architecture** | GitHub proxy + R2 fallback + minification | Pure R2 CDN with upload/browse UI |
| **URL Pattern** | `/:repo/:version/:file` | Direct `/:filepath` |
| **Features** | Minification, GitHub versioning, JSDelivr compat | Upload UI, file browser, analytics, delete, streaming |
| **Branding** | DXD CDN | Point CDN / PDC |
| **R2 Bucket** | `dxd-cdn` | `marketing-cdn` |

## Features to Add from marketing-cdn

1. **File Upload System** (`/upload` route)
   - Password-protected web form
   - Automatic file deduplication (-1, -2, etc.)
   - Success page with shareable URL

2. **File Browser** (`/browse` route)
   - Password-protected search interface
   - Fuzzy search functionality
   - Environment filters (staging/prod)
   - Folder filters
   - Copy URL / Copy HTML content
   - Context menu with stats and delete

3. **API Endpoints**
   - `/api/files` - List files with search/filter
   - `/api/file-stats` - Get file analytics
   - `/api/file-content` - Get HTML file content
   - `/api/delete-file` - Delete files

4. **Analytics Tracking**
   - Request count per file
   - First/last served timestamps
   - Stored in R2 at `analytics/{filepath}.json`

5. **MP4 Video Streaming**
   - Range request support
   - 1MB chunk optimization
   - Accept-Ranges headers

6. **CORS Configuration**
   - Configurable allowed origins (update for DXD domains)
   - Webflow branch detection pattern (update for DXD)

7. **Environment Variable**
   - `UPLOAD_PASSWORD` secret for protected routes

## Implementation Approach

### Option A: Hybrid Architecture (Recommended)
Keep dxd-cdn's GitHub proxy functionality AND add marketing-cdn features. This gives you:
- GitHub versioning and minification for code libraries
- Direct R2 upload/browse for marketing assets

The URL routing would be:
- `/upload`, `/browse`, `/api/*` тЖТ New marketing features
- `/:repo/:version/:file` тЖТ Existing GitHub proxy
- `/code/*` or other direct paths тЖТ Direct R2 serving (new)

### Option B: Replace Architecture
Replace dxd-cdn entirely with marketing-cdn architecture. You would lose:
- GitHub proxy/versioning
- On-the-fly minification
- JSDelivr URL conversion

## Decisions Made

1. **Architecture**: Hybrid - Keep GitHub proxy + minification AND add all marketing features
2. **CORS**: Allow all origins (`*`) for maximum flexibility
3. **Webflow Access**: Allow ALL `.webflow.io` domains (simplified from pattern matching)
4. **404 Handling**: Show error message instead of redirecting
5. **Directory Restriction**: Remove `code/` prefix - analytics/delete work on all client paths
6. **URL Structure**: `{client}/{project}/{env}/{path}` - no forced `code/` prefix

## GitHub Action Template

### File: `.github/workflows/deploy-to-cdn.yml`

This template will be created in the dxd-cdn repo and can be copied to client projects.

```yaml
name: Deploy to DXD CDN

on:
  push:
    branches:
      - main
      - staging
      - develop
    paths:
      - 'dist/**'
      - 'build/**'
      - 'src/**'
  workflow_dispatch:
    inputs:
      environment:
        description: 'Deployment environment'
        required: false
        type: choice
        options:
          - auto
          - staging
          - prod
        default: 'auto'

env:
  # ===== CONFIGURE THESE FOR EACH CLIENT PROJECT =====
  CLIENT_NAME: 'client-name'      # e.g., acme-corp
  PROJECT_NAME: 'project-name'    # e.g., marketing-site
  BUILD_DIR: 'dist'               # Directory containing built files
  # ===================================================

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Determine environment
        id: env
        run: |
          # Manual override takes precedence
          if [ "${{ github.event.inputs.environment }}" != "" ] && [ "${{ github.event.inputs.environment }}" != "auto" ]; then
            echo "environment=${{ github.event.inputs.environment }}" >> $GITHUB_OUTPUT
          # Branch-based detection
          elif [ "${{ github.ref_name }}" = "main" ] || [ "${{ github.ref_name }}" = "master" ]; then
            echo "environment=prod" >> $GITHUB_OUTPUT
          else
            echo "environment=staging" >> $GITHUB_OUTPUT
          fi
          
      - name: Display deployment info
        run: |
          echo "ЁЯЪА Deploying to DXD CDN"
          echo "   Client: ${{ env.CLIENT_NAME }}"
          echo "   Project: ${{ env.PROJECT_NAME }}"
          echo "   Environment: ${{ steps.env.outputs.environment }}"
          echo "   Branch: ${{ github.ref_name }}"
          echo "   Build Directory: ${{ env.BUILD_DIR }}"

      - name: Install rclone
        run: |
          curl https://rclone.org/install.sh | sudo bash

      - name: Configure rclone for Cloudflare R2
        run: |
          mkdir -p ~/.config/rclone
          cat > ~/.config/rclone/rclone.conf << EOF
          [r2]
          type = s3
          provider = Cloudflare
          access_key_id = ${{ secrets.R2_ACCESS_KEY_ID }}
          secret_access_key = ${{ secrets.R2_SECRET_ACCESS_KEY }}
          endpoint = https://${{ secrets.CLOUDFLARE_ACCOUNT_ID }}.r2.cloudflarestorage.com
          acl = private
          EOF

      - name: Sync files to R2
        run: |
          DEST_PATH="${{ env.CLIENT_NAME }}/${{ env.PROJECT_NAME }}/${{ steps.env.outputs.environment }}"
          echo "ЁЯУж Uploading to: $DEST_PATH"
          
          rclone sync ${{ env.BUILD_DIR }}/ r2:dxd-cdn/$DEST_PATH \
            --progress \
            --checksum \
            --verbose
          
          echo "тЬЕ Upload complete!"

      - name: Output CDN URLs
        run: |
          echo "## ЁЯФЧ CDN URLs" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "Base URL: \`https://cdn.designxdevelop.com/${{ env.CLIENT_NAME }}/${{ env.PROJECT_NAME }}/${{ steps.env.outputs.environment }}/\`" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "### Files uploaded:" >> $GITHUB_STEP_SUMMARY
          find ${{ env.BUILD_DIR }} -type f | while read file; do
            rel_path="${file#${{ env.BUILD_DIR }}/}"
            echo "- \`https://cdn.designxdevelop.com/${{ env.CLIENT_NAME }}/${{ env.PROJECT_NAME }}/${{ steps.env.outputs.environment }}/$rel_path\`" >> $GITHUB_STEP_SUMMARY
          done
```

function parseFilePath(filepath) {

    client: parts[0],
